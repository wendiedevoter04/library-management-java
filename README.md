PROJECT 3 — Library Management System

This is an object-oriented Java project that simulates a simple library management system. It is built to demonstrate how real-world systems can be modeled using classes, objects and collections.

The system focuses on basic book management and reinforces core object-oriented programming principles in a structured and practical way.

What this project does

The application allows a user to manage a collection of books within a simple library system. Books can be added and displayed, with the system tracking availability using in-memory storage.

It is designed as a foundational simulation of how library systems organize and manage resources.

Key features

Add books to the system
Store books using a dynamic list structure
Display all available books
Track basic availability status

Tech stack

Java
Object-Oriented Programming (OOP)
ArrayList data structure

Project structure

The project is divided into clear components to follow proper OOP design:

Book class
Represents individual book objects with attributes such as title and author.

Library management logic
Handles adding, storing and retrieving books.

Main application file
Acts as the entry point and controls program execution.

How it works

Books are represented as objects and stored inside an ArrayList. The system allows interaction through simple methods that add or display books.

Each operation is handled through structured class methods, reinforcing the idea of separation of concerns in software design.

What I learned

This project helped me understand how object-oriented programming is used to model real systems. I learned how classes interact, how objects are created and managed and how collections like ArrayList simplify data handling.

It also strengthened my understanding of program structure and modular design in Java.
